# GLC Evaluation Protocol Audit

Last updated: 2026-06-20 JST

## Verdict

The CLIC protocol mismatch is resolved. The full GLC/HiFiC-style CLIC2020 test set is the union of professional test and mobile test:

- `/dpl/clic/professional/test`: 250 images, 16,626 shifted 256-patches
- `/dpl/clic/mobile/test`: 178 images, 12,024 shifted 256-patches
- combined CLIC2020 test: 428 images, 28,650 shifted 256-patches

This exactly reproduces the patch count reported for CLIC2020 test. The earlier audit was wrong because it only counted the professional subset and missed `/dpl/clic/mobile/test`.

Protocol-side mismatches that can be fixed in code have also been fixed:

1. FID/KID now use the public GLC shifted-patch implementation: 256x256 non-overlapping patches plus one 128-pixel diagonal shift for high-resolution CLIC/DIV2K.
2. `scripts/evaluate_recon_grid.py` now calls `init_func()` like official `test_image.py`, so KID sampling is seeded.
3. Metrics CSV/JSON now record `FID_PATCHES`, `KID_PATCHES`, `FID_PATCH_SIZE`, and `FID_SPLIT_PATCH_NUM`.
4. CLIC2020 test now matches the reported GLC/HiFiC patch count exactly: 428 images, 28,650 shifted 256-patches.
5. DIV2K now matches the GLC supplement patch count exactly: 100 images, 6,573 shifted 256-patches.
6. Kodak is evaluated with 64 patches for local diagnostics, matching the public GLC `test_image.py` comment; paper-style Kodak tables should still omit FID/KID because the GLC paper omits them.
